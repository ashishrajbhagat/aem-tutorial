Page Editing and Tools

Touch UI Editor

The Touch UI (Touch-optimized UI) is the modern interface for content authors in AEM. It provides a responsive and user-friendly experience for editing and managing pages. The Touch UI editor is the default authoring interface in AEM and is preferred over the deprecated Classic UI.

Features of the Touch UI Editor:

  • Visual Editing: Allows authors to visually manage page layouts, text, images, and other components.
  • Side Panel: Includes tools such as the components browser, assets browser, and page structure tools.
  • Responsive Interface: Automatically adjusts to different screen sizes, making it useful on tablets and touch devices.
  • Quick Access: Easily access page properties, layout modes, and page information from the toolbar.

Components Browser

The Components Browser is a panel within the Touch UI editor that allows authors to drag and drop components onto the page. It contains all the allowed components as defined by the templateâ€™s policy.

Key features:

  • Component Groups: Components are categorized into groups like General, Layout, Form, and Custom.
  • Search: Quickly find a component using the search bar.
  • Allowed Components: Only the components permitted by the template are shown, maintaining design control and consistency.

This tool is vital for authors to build page content without writing any code.

Drag-and-Drop Editing

Drag-and-drop editing is the core of AEM's content authoring experience. Authors can position components by simply dragging them from the Components Browser and dropping them into the layout container or other component placeholders.

Highlights:

  • WYSIWYG Editing: See real-time layout and styling as components are placed and moved.
  • Interactive Controls: Edit text, images, links, and configurations directly within the dropped components.
  • Rearranging: Easily rearrange components by dragging them within or across containers.

This editing method empowers non-technical authors to build pages without developer assistance.

Responsive Grid System

The Responsive Grid (Layout Container) is a special container component in AEM that supports a flexible grid layout system. It enables authors to create responsive pages that adapt to different device sizes.

Key capabilities:

  • Column Layout: Layout containers allow authors to divide the page into responsive columns.
  • Custom Widths: Components can span multiple columns, and widths can be adjusted for desktop, tablet, and mobile views.
  • Breakpoints: The responsive grid adapts based on defined breakpoints, ensuring optimal display on all devices.
  • Drag Handles: Use drag handles to resize and reposition components within the grid.

This feature ensures that content looks great on any screen size, improving usability and accessibility.

In-Place Editing

In-place editing allows authors to modify content directly within the page view, without opening a separate dialog. Itâ€™s available for text, images, and some other component types.

Benefits of In-Place Editing:

  • Fast Editing: Quickly change headlines, body text, and image captions without disrupting workflow.
  • Real-Time Preview: Instantly see how the edited content appears on the actual page.
  • Fewer Dialogs: Reduces the need to open complex dialogs for simple edits.

It improves efficiency and is particularly useful for content editors working on marketing and blog pages.

Page Properties

Each AEM page has associated metadata and configuration settings, accessible via the Page Properties dialog. This includes:

  • Title & Name: Defines how the page appears in URLs and navigation.
  • Tags: Helps organize and classify content using taxonomy.
  • SEO Metadata: Includes description, keywords, and Open Graph settings for social sharing.
  • Language: Determines locale settings for multilingual sites.
  • Redirects: Setup redirect rules for deprecated or renamed pages.

Authors can access Page Properties by clicking the top-left Page Information icon and choosing "Page Properties" from the dropdown.
