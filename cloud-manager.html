Cloud Manager in AEM as a Cloud Service

Introduction to Cloud Manager

Cloud Manager is Adobe's CI/CD platform built specifically for managing and deploying code in AEM as a Cloud Service (AEMaaCS). It serves as the core DevOps interface, providing an automated, secure, and controlled way to deliver updates to AEM environments.

What is Cloud Manager?

  • A web-based tool provided by Adobe to manage AEM cloud environments.
  • Supports build, test, and deploy operations through customizable pipelines.
  • Integrated with Adobe Identity Management for access control and user roles.

Why Cloud Manager is Central to AEMaaCS

  • AEMaaCS does not allow manual deployments to production environments.
  • All code deployments must go through Cloud Manager pipelines.
  • Ensures consistency, automation, and security across all environments.

Role in DevOps and CI/CD Automation

  • Implements automated build, test, quality check, and deploy steps.
  • Reduces human errors by enforcing predefined steps and policies.
  • Supports DevOps principles like continuous integration and continuous delivery.

Key Features and Benefits

  • Production and non-production pipelines
  • Automated performance and security testing
  • Role-based deployment approvals
  • Integrated quality gates and code validation
  • Visibility into logs, metrics, and environment health

Pipelines in Cloud Manager

Pipelines in Cloud Manager orchestrate the automated flow of building, validating, and deploying code into AEM cloud environments. There are different types of pipelines based on the target environment and control levels.

Types of Pipelines

  • Non-Production Pipeline: Used for development and stage environments. These pipelines deploy automatically without manual approvals.
  • Production Pipeline: Includes additional quality gates and a mandatory manual approval step before deploying to production.

CI/CD Workflow in Cloud Manager

Each pipeline includes a series of automated steps:

  1. Code Build: Compiles and packages code using Adobe's Cloud SDK and Maven tooling.
  2. Code Quality: Performs static analysis using Adobe-defined rules (e.g., code smells, maintainability).
  3. Security Scans: Scans for known vulnerabilities in third-party dependencies.
  4. Performance Testing: Runs automated tests in a stage environment to simulate traffic and measure performance.
  5. Deployment: Deploys code to the designated AEM environment (with optional manual approval).

Custom Pipeline Variables and Configuration

  • Define environment variables, secrets, and flags for custom behaviors.
  • Set up filters to trigger deployments only for specific branches or projects.
  • Use advanced pipeline YAML templates (if supported by Adobe) to control execution logic.

Environments

In AEM as a Cloud Service, environments represent isolated instances of AEM used for different stages of the development lifecycle. These include development, staging, and production environments, all managed through Cloud Manager.

Default AEMaaCS Environments

  • Development: Used by developers to test features, integrations, and content workflows.
  • Stage: Mirrors production; used for UAT, performance testing, and release validation.
  • Production: The live customer-facing environment. Requires strict control and approval for deployments.

Managing Environments

Code Quality and Static Analysis

Cloud Manager enforces code quality through automated static analysis during the build process. This helps prevent issues from reaching production and ensures adherence to Adobe's AEM best practices.

What Is Checked

  • Maintainability: Ensures clean, maintainable code (e.g., unused variables, cyclomatic complexity).
  • Security: Checks for vulnerabilities in custom code and third-party libraries (e.g., CVEs).
  • Performance: Warns against performance bottlenecks like synchronous resource calls or inefficient queries.
  • Correctness: Validates common AEM patterns like resource resolution, service injection, and configuration management.

Quality Gates

  • Each code scan is scored (0-100) and compared against thresholds.
  • Failing a quality gate (e.g., below 60) stops deployment to stage or production.
  • Thresholds and scoring logic are based on Adobe's internal rule sets.

Suppressions and Customization

  • You can suppress certain rules using inline comments or configuration files, if justified.
  • Custom rules can't be added directly, but code should be adjusted to conform to Cloud Manager's standards.

Reporting

  • Static analysis reports are downloadable from the Cloud Manager pipeline execution screen.
  • Developers can review the rule breakdown, score summary, and file-level issues.
  • Helps track technical debt over time.

Performance Testing

Cloud Manager includes built-in performance testing as part of the production pipeline to ensure that code changes do not negatively impact the stability and speed of the AEM site under load.

Purpose of Performance Testing

  • Simulate real-world load conditions on the staging environment.
  • Measure key performance indicators (KPIs) before deployment.
  • Catch issues like slow response times or memory spikes early.

Performance Metrics Captured

  • Response Time: Average and peak times during load simulation.
  • Error Rate: Number of failed requests or exceptions.
  • Throughput: Requests per second the system can handle.
  • CPU/Memory Usage: System resource utilization during test run.

How It Works

  • Automatically runs during the stage deployment phase of the production pipeline.
  • Load tests simulate anonymous and logged-in user traffic patterns.
  • Results determine whether the pipeline proceeds or fails.

Thresholds and Failures

  • If thresholds are breached (e.g., high error rate), deployment to production is blocked.
  • Detailed test reports are available in the pipeline logs for investigation.

Deployment Approval

For production environments, Cloud Manager enforces a manual approval step after successful code validation and testing. This ensures human oversight before live deployment.

Manual Approval Workflow

  • Once a production pipeline passes all quality gates, performance tests, and validations, it pauses for approval.
  • Designated users must log in to Cloud Manager to approve or reject the deployment.
  • Approval must occur within a specific time window or the pipeline expires.

Who Can Approve?

  • Roles like Business Owner or Program Manager have access to the approval step.
  • Roles are assigned via the Adobe Admin Console.

Deployment Freeze and Governance

  • Cloud Manager allows defining blackout periods during which deployments are blocked.
  • Helps align with business release calendars and compliance requirements.

Best Practices

  • Always review test results and static analysis before approving.
  • Notify stakeholders when a deployment is about to go live.
  • Ensure rollback plan is in place in case of post-deployment issues.

Access and Permissions

Cloud Manager uses role-based access control (RBAC) managed through the Adobe Admin Console. Each user role has predefined capabilities, ensuring security and accountability during deployments and environment management.

Cloud Manager Roles

  • Developer: Can view logs, execute non-prod pipelines, and troubleshoot issues.
  • Program Manager: Full access to pipeline and environment configurations.
  • Deployment Manager: Can run and approve production deployments.
  • Business Owner: Can approve production deployments; often involved in final go-live decisions.

Managing Roles

  • Roles are assigned via the Adobe Admin Console under your organization.
  • Multiple users can be assigned to each role for team-based access control.
  • Permissions propagate across all environments and programs assigned to that user.

Monitoring and Logs

Cloud Manager provides tools for monitoring pipeline execution, environment status, and system health. Logs are accessible in real-time for both developers and deployment managers.

Pipeline Logs

  • Each pipeline step (build, test, deploy) produces downloadable logs.
  • Logs include static analysis details, test results, and error stacks if any step fails.
  • Available via the Cloud Manager UI ‚Üí Pipeline Execution tab.

Environment Monitoring

  • Access CPU, memory, disk, and request metrics for each environment.
  • Helps diagnose performance bottlenecks or unusual behavior.
  • Particularly useful when reviewing test results or failed deployments.

Health and Alerts

  • AEMaaCS continuously monitors environment uptime and application health.
  • Admins receive alerts via the Adobe system when critical issues are detected.

Best Practices

Following Cloud Manager best practices ensures smoother deployments, greater consistency, and better team collaboration.

Pipeline & Deployment Practices

  • Use non-prod pipelines regularly to validate feature branches early.
  • Never bypass the production pipeline; use it to catch regressions.
  • Keep production pipeline approvals restricted to responsible stakeholders.

Code Quality & Testing

  • Address all code quality issues before attempting a production deployment.
  • Use the static analysis report to improve maintainability and security.
  • Fail fast ‚Äî treat quality gates and performance tests as blocking conditions.

Configuration & Secrets

  • Always separate dev, stage, and prod configurations using runmode folders.
  • Never hardcode secrets ‚Äî use the appropriate secure storage mechanisms.

Team & Role Management

  • Assign least privilege access ‚Äî only give deployment rights to trusted roles.
  • Review and audit user roles periodically in Adobe Admin Console.
  • Enable backup approvers to avoid production pipeline delays.

Summary

Cloud Manager is the cornerstone of DevOps in AEM as a Cloud Service. It enables secure, automated, and reliable deployments through a streamlined CI/CD pipeline that incorporates best practices like static code analysis, performance testing, and manual approvals for production releases.

By leveraging Cloud Manager's environment management, role-based access, and built-in quality gates, teams can ensure smoother development cycles, faster go-to-market timelines, and improved code quality. With proper pipeline setup and adherence to best practices, Cloud Manager empowers organizations to confidently deploy and scale their digital experiences in the cloud.
