Tagging and Metadata Management

Introduction

Tagging in Adobe Experience Manager (AEM) is a powerful way to organize, classify, and retrieve content efficiently. Tags provide a structured taxonomy that helps both authors and systems identify the intent, topic, or purpose of a given piece of content or asset.

Purpose of Tagging in AEM

  • Enable content classification and categorization
  • Support personalized content delivery and targeting
  • Improve internal searchability for authors and marketers
  • Enhance reporting and analytics through segment-based tagging

What Are Tags in AEM?

Tags in AEM are hierarchical metadata objects used to classify content, assets, and pages. They are defined in a structured format and can be applied consistently across different parts of the platform.

AEM Tag Structure

  • Tags follow the format: namespace:tag-name
  • Hierarchical tags appear like: industry:technology/ai

Where Tags Are Stored

  • All tags are stored under /etc/tags in the AEM JCR repository
  • Each tag is a node with properties such as jcr:title, cq:tags, etc.

Tags vs. Keywords vs. Metadata

  • Tags: Structured, hierarchical, and reusable taxonomy items
  • Keywords: Free-form text often added as metadata fields, less structured
  • Metadata: Broader category that includes tags, descriptions, titles, etc.

Tag Namespaces

Namespaces allow organizations to logically group their tags under meaningful categories. This helps prevent naming conflicts and improves tag discoverability.

Default Namespaces in AEM

  • standard: Common tags provided out-of-the-box
  • geometrixx: Sample namespace from demo content

Creating Custom Namespaces

  • Navigate to Tools → General → Tagging in Touch UI
  • Create a new folder (namespace) and define tags within it
  • Use meaningful and consistent names (e.g., industry, region)

Best Practices for Organizing Taxonomy

  • Keep namespaces broad and tags specific
  • Avoid creating too many flat, unrelated tags
  • Use a consistent naming convention (e.g., lowercase, hyphenated)
  • Periodically review and clean unused or duplicate tags

Creating and Managing Tags

Tags in AEM can be easily created and organized using the Touch UI Tag Console. Authors or administrators with appropriate permissions can manage the taxonomy in a hierarchical structure.

Using Touch UI Tag Manager

  • Navigate to Tools → General → Tagging.
  • Use the interface to:
    • Create new namespaces (folders)
    • Add tags under each namespace
    • Edit tag titles or move tags to different hierarchies
    • Delete unused or incorrect tags

Tag Hierarchy and Organization

  • Tags can have nested sub-tags to represent hierarchies (e.g., topic/technology/ai).
  • Each tag is stored as a node under /etc/tags.

Permissions for Tag Management

  • Only users with modify rights on /etc/tags can create or delete tags.
  • Restrict access to prevent unauthorized or duplicate tag creation.

Tag Properties

  • jcr:title: Display name of the tag
  • jcr:description: Optional description for internal use
  • sling:resourceType: Usually set to cq/tagging/components/tag

Using Tags in Components

Tags are commonly used in AEM components to categorize content, enable search filters, and drive personalization logic. They are stored as part of the component's content node under the cq:tags property.

Applying Tags to a Page or Component

  • Open the page in Edit mode and go to Page Properties → Basic.
  • Use the Tags field to assign one or more tags to the page.
  • For components, authors can select tags via dialog fields configured with tag input types.

cq:tags Property

  • Stored as a multi-valued String array on the JCR node (e.g., ["industry:finance", "region:asia"]).
  • Used internally for classification, filtering, and search queries.

Use Case Examples

  • Article or Blog Pages: Categorize by topic, industry, or region.
  • Product Listings: Filter by category tags.
  • Personalization: Target segments based on tagged content.

Tags in Editable Templates

  • You can enforce tag-based logic in content policies or structure components.
  • For example, restrict certain templates to content tagged under specific namespaces.

Tags in Assets

Tags play a crucial role in organizing and classifying digital assets within AEM Assets. They improve discoverability, enable filtered searches, and support automation and personalization workflows.

Tagging Digital Assets

  • Open the Assets console and select an image, video, or document.
  • Click Properties → Basic tab → Add tags using the Tags field.
  • Tags are stored under the cq:tags property on the asset metadata node.

Bulk Tagging

  • Use the Assets Console → Select multiple assets → Click Properties.
  • Add or modify tags for multiple assets at once.
  • Useful during onboarding of large media libraries.

Smart Tags (AI-based Auto-Tagging)

  • AEM can use Adobe Sensei to generate tags automatically based on image analysis.
  • Tags are added to cq:tags but marked with smartTags metadata for distinction.
  • Smart tagging must be explicitly enabled and configured.

Search and Filter by Tags

  • Tags appear in the left-hand facet filters in the Assets console.
  • Users can filter by tags to quickly find related assets.

Metadata Schemas

Metadata Schemas in AEM define the structure of asset metadata forms. You can customize which fields appear for asset folders, including tag fields, required fields, dropdowns, and more.

What is a Metadata Schema?

  • A configuration that controls how asset metadata fields are displayed and edited in AEM.
  • Assigned at the folder level and used when editing asset properties.

Applying Schemas to Folders

  • Go to Tools → Assets → Metadata Schema.
  • Select or create a schema form (e.g., default, marketing).
  • Assign it to an asset folder from the folder's properties screen.

Adding Tag Fields to Schema

  • Drag a Tags field from the left panel into the schema editor.
  • Bind it to the cq:tags property or another custom metadata path.
  • Set constraints like required/optional, namespace restrictions, or suggestions.

Example Use Cases

  • Add a “Project Type” tag field to all assets in the campaign folder.
  • Create a “Region” field with dropdown values pulled from tag namespaces.
  • Require at least one classification tag before an asset can be published.

Programmatic Access to Tags

AEM provides APIs that allow developers to create, retrieve, and assign tags programmatically. This is useful when integrating external systems, automating tagging, or processing content dynamically.

Using TagManager API (Java/Sling)

  • Obtain the TagManager from the ResourceResolver: 
    TagManager tagManager = resolver.adaptTo(TagManager.class);
  • Create or resolve a tag: 
    Tag tag = tagManager.resolve("my-namespace:my-tag");
  • Assign tags to a resource: 
    tagManager.setTags(resource, new Tag[]{tag}, true);

Accessing cq:tags in Sling Models

  • Inject tags using: 
    @ValueMapValue
private String[] cqTags;
  • Optionally convert tag IDs to titles via TagManager.

Querying Tagged Content

  • Use QueryBuilder to search by tag: 
    path=/content
property=cq:tags
property.value=industry:finance
  • Or use GraphQL/Content Fragment API to filter by tag fields.

Use Cases

  • Auto-tag content during workflow or import
  • Tag assets based on metadata rules
  • Bulk-update tags via custom servlets

Search, Filtering, and Personalization

Tags significantly enhance how users find, sort, and interact with content. They support search and filtering UIs as well as dynamic content delivery strategies.

Faceted Search

  • Use tags as filters in search components (e.g., blogs, products, assets).
  • Populate dropdowns or checkboxes with available tags via Java or GraphQL.
  • Faceted navigation improves UX by narrowing results by topic or category.

Tag-Based Filtering

  • Use tag queries in Sling Models, Servlets, or GraphQL to show only relevant content.
  • Example: Filter news articles tagged with topic:economy.

Personalization Based on Tags

  • Target users based on content tags via ContextHub or Adobe Target.
  • Serve different banners or articles based on user interests (e.g., sports, finance).
  • Combine tags with user profiles or segments for deeper targeting.

Real-World Examples

  • Filter recipes on a food blog using cuisine and ingredient tags.
  • Personalize homepage content based on region or product interest tags.
  • Create a dynamic search interface for assets by brand, usage type, and season.

Best Practices

Implementing a consistent and scalable tagging strategy in AEM is essential for long-term success. Below are key recommendations for managing tags effectively across content and assets:

Plan Before You Tag

  • Design a clear and structured taxonomy before content creation begins.
  • Group tags into logical namespaces (e.g., industry, topic, region).
  • Document the purpose and usage of each tag group for teams.

Prefer Tags Over Free-Form Metadata

  • Use AEM's cq:tags property instead of unstructured keyword fields.
  • Tags offer validation, hierarchy, localization, and reuse capabilities.

Use Namespaces Consistently

  • Do not mix unrelated tags under a single namespace.
  • Stick to lowercase and hyphenated naming conventions (e.g., topic:machine-learning).

Restrict Who Can Create Tags

  • Limit tag creation to a small group of admins or content strategists.
  • Prevent taxonomy pollution from user-generated or redundant tags.

Clean and Audit Tags Regularly

  • Remove unused, duplicated, or obsolete tags from /etc/tags.
  • Use reporting tools or scripts to identify stale or unreferenced tags.

Test Tagging Experience

  • Ensure authors can easily search and apply tags in components and metadata dialogs.
  • Use autocomplete and tag filters in dialogs to improve usability.

Integrate Tags into Search and Personalization

  • Enable faceted search on sites and asset portals using tags.
  • Use tags to personalize content delivery with ContextHub or Adobe Target.

Avoid Hardcoding Tags

  • Do not hardcode tag values in components or queries.
  • Use dynamic fetching and configuration to maintain flexibility.

Summary

Tagging and metadata management in AEM play a foundational role in organizing content, improving author efficiency, enabling personalization, and powering search and analytics. A well-structured taxonomy ensures consistency and scalability across projects, while integration with components, assets, and search tools enhances the overall digital experience.

By combining thoughtful taxonomy design, clear authoring practices, and programmatic control, AEM users can unlock the full potential of tags across Sites and Assets. Whether you're an author, developer, or administrator, mastering tag and metadata strategies is key to managing content effectively in AEM.
