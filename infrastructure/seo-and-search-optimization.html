SEO & Search Optimization

Search Engine Optimization in AEM

SEO (Search Engine Optimization) is critical to ensure that your AEM-powered website is discoverable by search engines like Google and Bing. AEM provides powerful tools and flexibility to implement both on-page and technical SEO best practices.

Importance of SEO for AEM Sites

  • Improves organic visibility and traffic
  • Supports multilingual and multisite SEO strategies
  • Helps optimize performance, accessibility, and mobile readiness
  • Enables structured metadata, improving click-through rates (CTR)

On-page vs. Technical SEO

  • On-page SEO: Deals with title tags, meta descriptions, headings, content quality, images, alt texts, etc.
  • Technical SEO: Involves sitemaps, robots.txt, canonical tags, page speed, structured data, and mobile-first optimization.

Sitemap Generation

Sitemaps help search engines crawl your site effectively. AEM can generate sitemaps dynamically using custom logic or through tools like ACS AEM Commons Sitemap Servlet.

Enabling sitemap.xml in AEM

To generate sitemaps in AEM:

  • Install ACS AEM Commons (if not already installed)
  • Create a Sitemap configuration under /conf or directly via the Sitemap Manager UI
  • Access the sitemap at: /content/your-site.sitemap.xml

Configuring Path Filters

You can include or exclude specific paths from the sitemap using configuration:

"includePaths": [
  "/content/my-site/en",
  "/content/my-site/fr"
],
"excludePaths": [
  "/content/my-site/en/hidden",
  "/content/my-site/en/error"
]

Auto-generating Multilingual Sitemaps

If your AEM site supports multiple languages using Language Copies or Live Copies, you can auto-generate separate sitemaps or use hreflang annotations for each locale.

Example:

<url>
  <loc>https://www.example.com/en/product.html</loc>
  <xhtml:link rel="alternate" hreflang="fr" href="https://www.example.com/fr/product.html"/>
  <xhtml:link rel="alternate" hreflang="es" href="https://www.example.com/es/product.html"/>
</url>

robots.txt Management

The robots.txt file tells search engine crawlers which parts of your site should or should not be indexed. It's a critical part of your technical SEO setup in AEM.

Where to Place robots.txt in AEM

  • Place it under /content/your-site/robots.txt and configure a dispatcher rule to serve it as a flat file.
  • Alternatively, you can create a specific Sling mapping at /etc/robots.txt and expose it as a system resource.
  • Ensure the file is accessible at the root: https://www.example.com/robots.txt

Dispatcher-Level Caching Considerations

Since robots.txt rarely changes, cache it at the dispatcher level to improve performance and reduce unnecessary AEM hits.

/cache {
  /rules {
    /0001 { /type "allow" /url "/robots.txt" }
  }
}

Allow/Disallow Rules (Examples)

User-agent: *
Disallow: /content/secure/
Disallow: /crx/
Allow: /content/public/

You can also create different rules for different crawlers if needed:

User-agent: Googlebot
Disallow: /private/

User-agent: Bingbot
Disallow: /admin/

Meta Tags and Canonical URLs

Properly setting page metadata and canonical URLs ensures that search engines understand your page content and avoid duplicate content issues.

Setting <title> and <meta> Tags Dynamically

  • Use the AEM Page Properties (like jcr:title, dc:description, cq:tags) to populate metadata
  • Inject them dynamically in the page head using HTL or Sling Models

Using cq:htmlTag and Page Properties

Use the cq:htmlTag property or custom components to dynamically include meta tags:

<head>
  <title>${currentPage.pageTitle} - My Site</title>
  <meta name="description" content="${currentPage.description}" />
  <meta name="keywords" content="${currentPage['cq:tags']}" />
</head>

Handling Canonical URLs for SEO

  • Use canonical tags to indicate the preferred version of a page
  • Helpful for multilingual, multisite, or faceted navigation setups
<link rel="canonical" href="https://www.example.com${currentPage.path}.html"/>

Structured Data (Schema.org Markup)

Structured data helps search engines understand your content better. AEM sites can embed JSON-LD using custom components or via the page header.

JSON-LD Script for Products, Articles, etc.

Use JSON-LD (preferred by Google) to describe structured content like products, articles, breadcrumbs, and more:

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Wireless Headphones",
  "image": "https://www.example.com/images/headphones.jpg",
  "description": "High-quality wireless headphones with noise cancellation.",
  "sku": "WH-12345",
  "offers": {
    "@type": "Offer",
    "priceCurrency": "USD",
    "price": "99.99",
    "availability": "https://schema.org/InStock"
  }
}
</script>

Embedding via Custom Components

  • Create a component (e.g., structured-data) and bind values via Sling Models
  • Include it in the page header or in relevant page templates
  • Ensure the output is valid using Google's Rich Results Test

Search Engine Integration

Integrating with search engines allows you to monitor indexing, submit sitemaps, and improve discoverability.

Google Search Console Verification

  • Add a meta tag to the page header:
    • <meta name="google-site-verification" content="your-verification-code" />
  • Or upload the verification file to the DAM or Dispatcher public directory and serve it from the root.
  • Once verified, submit your sitemap URL (e.g., https://www.example.com/sitemap.xml) via the Search Console.

Bing Webmaster Integration

  • Similar to Google, add the meta verification tag:
    • <meta name="msvalidate.01" content="bing-verification-code" />
  • Or use the file-based method for verification.
  • Submit your sitemap URL in the Bing Webmaster Tools dashboard.

SEO Best Practices

  • Avoid Duplicate Content: Use canonical URLs, redirects, and proper hreflang tags.
  • SEO-Friendly URLs: Keep URLs clean and readable, avoid unnecessary extensions, and use Dispatcher rewrites.
  • Mobile-First: Ensure responsive design, use viewport meta tag, and optimize for mobile speed.
